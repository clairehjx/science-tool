Science Cheat Sheet Pipeline

Step-by-step setup guide — Singapore P4/P5 Science exam study tool

Prerequisites
1
Install Python 3.11 or later

Install Homebrew if you don't have it, then install Python:

# Install Homebrew (skip if already installed)
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# Install Python
brew install python@3.12
Verify with python3 --version — should show 3.11 or higher.

On Ubuntu / Debian:

sudo apt update && sudo apt install python3 python3-pip python3-venv -y

On Fedora / RHEL:

sudo dnf install python3 python3-pip -y
Verify with python3 --version — should show 3.11 or higher.
2
Get the project files

Copy the project folder to your machine. The folder should look like this:

science_cheatsheet/
  CLAUDE.md
  requirements.txt
  banks/p4/papers/   # drop P4 PDFs here
  banks/p5/papers/   # drop P5 PDFs here
  conceptviz/         # ConceptViz PNGs (canonical + v1/v2/v3 variants)
  output/html/       # final cheat sheet HTML
  output/validated/   # cached validated questions JSON
  output/conceptviz_prompts/   # ConceptViz structured content (.txt)
  prompts/
  scripts/
  templates/

Open a terminal and navigate into the project folder:

cd ~/science_cheatsheet
3
Create a virtual environment

Run once to create an isolated Python environment inside the project folder:

python3 -m venv venv

Activate it (you need to do this every time you open a new terminal):

source venv/bin/activate
Your prompt will show (venv) when the environment is active.
4
Install dependencies
pip install -r requirements.txt
The first run also downloads the sentence-transformer model (~80 MB) automatically when you first run the pipeline. This only happens once.
5
Add your Gemini API key

Set your Gemini API key as an environment variable. On Mac/Linux, add this line to your shell profile (such as ~/.zshrc or ~/.bashrc), then restart your terminal:

export GEMINI_API_KEY=your_key_here

Replace your_key_here with your actual key from Google AI Studio.

The key lives in your computer's environment, never inside the project folder — so it can never be committed to git by accident.
Running the pipeline
6
Add exam PDFs

Copy your exam paper PDFs into the correct bank folder. File names must follow this format:

# Format: School_Year_Level.pdf
banks/p5/papers/ACS_Junior_2025_P5.pdf
banks/p5/papers/Raffles_Girls_2024_P5.pdf
banks/p4/papers/Henry_Park_2025_P4.pdf
Multi-word school names use underscores. The year and level are extracted automatically from the filename.
7
Extract questions from PDFs

Run the extractor for your bank. It skips PDFs already processed — safe to re-run.

# For P5 papers
venv/bin/python3 scripts/pdf_extractor.py --bank p5

# For P4 papers
venv/bin/python3 scripts/pdf_extractor.py --bank p4
This makes 1 Gemini API call per new PDF. Already-processed PDFs are skipped automatically (tracked in processed.json).
8
Generate a cheat sheet

Run the pipeline for a topic. Use the exact topic name as it appears in the syllabus.

venv/bin/python3 scripts/pipeline.py --topic "Cycles in Water" --bank p5

Useful flags:

FlagWhat it does
--threshold 0.20Lower the similarity filter if fewer than 5 questions match (default 0.25)
--force-validationRe-run Gemini validation even if a saved result already exists
--force-htmlRegenerate the HTML even if it already exists (otherwise skipped)
--wide-themeSplit by sub-topic, produce one HTML file per sub-topic
First run makes 2 Gemini API calls: validation + HTML generation. Re-runs skip both automatically if their outputs exist (use the --force-* flags to override).
Outputs saved to:
• output/html/cheatsheet_cycles_in_water_p5.html — the cheat sheet (inline SVGs, tables, charts already rendered)
• output/validated/validated_cycles_in_water_p5.json — cached validated questions
• output/conceptviz_prompts/cycles_in_water_p5.txt — structured ConceptViz content (consumed by image_generator)
Compact helper DSL. The HTML LLM emits short helper calls (kCard, mCard, tCard, dCardCompare for cards; dCompare, dDistinct, dScene, dFlow for SVG diagrams; ecBar, ecLine, ecPie for charts) instead of raw HTML/SVG. Helpers live in templates/script.js; each tab body is a small <script> block calling them. Topic-specific Tier-2 sub-shapes (water/heat/matter, biology, electricity) are selectively injected into the prompt by pipeline.py based on keyword match — the LLM only sees the helpers relevant to the topic.
9
Generate the ConceptViz + any PNG diagrams

Most diagrams (tables, charts, Rough.js sketches, complex SVGs) are already rendered inline by the HTML generator in step 8. image_generator.py only handles the parts that need image-model calls:

  • ConceptViz (Master Summary tab) — runs a 3-stage flow: prompt-shaper LLM → N nano-banana variants → embed v1
  • Nano-banana PNG placeholders — rare; only when the HTML LLM picked TYPE D-PNG for genuinely photorealistic content
# Interactive — shows a menu to choose which tab to generate
venv/bin/python3 scripts/image_generator.py --topic "Cycles in Water" --bank p5

# Only the Master Summary (ConceptViz)
venv/bin/python3 scripts/image_generator.py --topic "Cycles in Water" --bank p5 --tab summary

# Generate more ConceptViz variants (default 3)
venv/bin/python3 scripts/image_generator.py --topic "Cycles in Water" --bank p5 --variants 5
Already-replaced placeholders are skipped automatically. ConceptViz also skips if conceptviz/{slug}_{level}.png already exists — embeds the file directly with no API calls.
10
Open the cheat sheet
open output/html/cheatsheet_cycles_in_water_p5.html
xdg-open output/html/cheatsheet_cycles_in_water_p5.html
The cheat sheet is a single self-contained HTML file — no internet connection needed to view it. Share it by copying the file.
ConceptViz (Master Summary diagram)
★
How ConceptViz is generated

The Master Summary tab shows a full-screen illustrated concept map. It's now generated automatically as part of step 9 in three stages:

  1. Stage 1 — content extraction (already done in step 8). The HTML LLM mines the validated questions and writes structured YAML content (sub-topics, keywords, Q&A pairs, causal pairs, classification, key takeaway, scene hint) into the hidden ConceptViz placeholder div. Same content also dumped to output/conceptviz_prompts/{slug}_{level}.txt.
  2. Stage 2 — prompt shaping. image_generator.py calls a small flash LLM to combine that structured content with templates/conceptviz_image_style.txt, returning N variant image prompts.
  3. Stage 3 — image rendering. Each variant prompt is sent to nano-banana-2 (gemini-3.1-flash-image-preview). All variants are saved alongside their prompts; v1 is promoted to the canonical filename and embedded in the HTML.

Files produced (default --variants 3):

conceptviz/cycles_in_water_p5.png   # canonical (= v1, embedded into HTML)
conceptviz/cycles_in_water_p5_v1.png   # variant 1
conceptviz/cycles_in_water_p5_v1.prompt.txt
conceptviz/cycles_in_water_p5_v2.png   # variant 2
conceptviz/cycles_in_water_p5_v2.prompt.txt
conceptviz/cycles_in_water_p5_v3.png   # variant 3
conceptviz/cycles_in_water_p5_v3.prompt.txt

Swap to a different variant: copy any _v{N}.png over the canonical filename, then re-run image_generator.py — it embeds the existing PNG without making any API calls.

cp conceptviz/cycles_in_water_p5_v2.png conceptviz/cycles_in_water_p5.png
venv/bin/python3 scripts/image_generator.py --topic "Cycles in Water" --bank p5 --tab summary
If you'd rather generate the PNG with an external tool, just save it to conceptviz/{slug}_{level}.png directly — image_generator detects the existing file and skips Stage 2 + 3 entirely.
Growing the helper catalogue
↻
Add new Tier-2 sub-shapes after each run

Each topic the LLM may fall back to raw <svg> for shapes the catalogue doesn't yet cover (a magnifying glass, a magnet, a syringe…). Use this loop to grow the catalogue from real examples:

  1. Run the pipeline and open the HTML.
  2. Open prompts/iterate_tier2_helpers.txt and copy the prompt block (between the marked lines).
  3. Paste it into a fresh Claude conversation, fill in {SLUG} and {LEVEL}, and attach the new HTML.
  4. Claude scans the HTML for raw SVG blocks, sources clean implementations from bioicons (CC0) or SVGRepo (CC0/MIT), and returns a drop-in patch for templates/script.js plus a worked example for prompts/generation_prompt.txt.
  5. Apply the patch and re-run the pipeline with --force-html on the same topic.
The iteration prompt is version-controlled at prompts/iterate_tier2_helpers.txt — reuse the same text every cycle so future helpers stay consistent with the existing API style (camelCase d… names, dPal palette, pure string returns).
